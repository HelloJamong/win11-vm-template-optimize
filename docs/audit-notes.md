# 공공기관/망분리 환경 Audit Mode 체크포인트

이 문서는 Windows 11 VM 마스터 템플릿을 공공기관 또는 망분리 환경에 배포하기 전 Audit Mode에서 확인해야 할 체크포인트를 정리합니다.

## 1. 기본 원칙

- 조직 보안 정책과 감사 기준이 이 문서보다 우선합니다.
- 불필요 기능 제거는 “업무 영향도 검토 후 적용”을 원칙으로 합니다.
- 삭제보다 비활성화가 안전한 항목은 비활성화를 우선 검토합니다.
- 실행한 정리/삭제/비활성화 작업은 변경 이력에 기록합니다.
- 템플릿 최종화 전 복제 VM에서 업무 앱, 보안 에이전트, 업데이트 흐름을 검증합니다.

## 2. Administrator 계정/폴더 처리

확인 항목:

- Administrator 바탕 화면, 다운로드, 문서 폴더에 작업 파일이 남아 있지 않은지 확인
- `%TEMP%` 및 `C:\Users\Administrator\AppData\Local\Temp` 정리
- 브라우저 다운로드 기록, 캐시, 자동 완성 데이터 정리
- Administrator 계정 활성/비활성 정책 확인
- 기본 비밀번호 또는 임시 비밀번호가 이미지에 남지 않도록 관리

감사 대응 포인트:

- Audit Mode 작업 계정으로 Administrator를 사용한 사유 기록
- 계정 상태 변경 여부 기록
- 삭제한 작업 파일 목록 또는 보존 위치 기록

## 3. defaultuser0 처리

확인 항목:

- 기본 OS 프로필 경로 또는 별도 사용자 프로필 드라이브 아래의 `defaultuser0` 존재 여부
- 로컬 사용자 목록에 `defaultuser0` 잔여 여부
- OOBE 완료 전 임시 상태인지 여부

주의사항:

- Sysprep 전 단계에서 무리하게 삭제하지 않습니다.
- OOBE 테스트 후 잔여 계정으로 확인될 때 정리 절차를 검토합니다.
- 삭제 전 복제 VM에서 영향을 확인합니다.

## 4. Phone Link, Xbox, Copilot 제거

공공기관/망분리 환경에서는 다음 앱/기능이 불필요하거나 제한 대상일 수 있습니다.

- Phone Link
- Xbox 관련 앱
- Copilot 관련 앱 또는 패키지
- Teams 개인용 구성요소
- 소비자 경험 기반 추천 앱

운영 기준:

- 제거 후보는 `configs/appx-remove-list.txt`에서 관리합니다.
- 제거 전 업무 앱 의존성을 확인합니다.
- 제거 실패 항목은 로그로 남기되 전체 작업을 중단하지 않습니다.
- Provisioned Appx 제거와 현재 사용자 Appx 제거를 구분합니다.

감사 대응 포인트:

- 제거 근거: 망분리, 불필요 클라우드 연동, 소비자 기능 차단
- 제거 범위: 현재 사용자, 전체 사용자, provisioned package 여부
- 업무 영향도 검증 결과

## 5. 검색/Bing/클라우드 검색 차단

검토 정책:

- 웹 검색 비활성화
- Bing 검색 통합 제한
- 클라우드 콘텐츠 검색 제한
- 검색 하이라이트 제한
- Cortana 관련 정책 비활성화

주의사항:

- Windows 버전별 정책 키 적용 범위가 다를 수 있습니다.
- 검색 품질이나 시작 메뉴 검색 동작이 일부 달라질 수 있습니다.
- 조직 GPO가 있는 경우 로컬 정책보다 GPO가 우선할 수 있습니다.

## 6. Recall/Copilot/Consumer/privacy 정책

Windows 11 버전과 에디션에 따라 제공 여부가 다를 수 있으나, 템플릿 기준에서는 다음 방향을 권장합니다.

- Copilot 기능 제한
- Recall 또는 스냅샷성 기능 제한
- 소비자 경험 기능 비활성화
- 광고 ID 및 맞춤형 추천 제한
- 진단 데이터 최소화 정책 검토
- 앱 시작 추적 제한
- 위치, 음성, 필기 입력 데이터 수집 정책 검토

감사 대응 포인트:

- 개인정보 처리 최소화 원칙 적용 여부
- 클라우드 연동 기능 제한 여부
- 정책 적용 경로와 적용 일자 기록

## 7. 로그/캐시 최소화

정리 후보:

- Windows Temp
- 사용자 Temp
- Windows Update Download cache
- Defender Scans History
- Windows Error Reporting
- CrashDumps
- 브라우저/웹뷰 캐시
- 이벤트 로그

주의사항:

- 이벤트 로그 삭제는 감사 목적과 충돌할 수 있습니다.
- 장애 분석이 필요한 로그는 삭제 전 보존합니다.
- 최종 템플릿 직전 정리한 경우 실행자, 일시, 범위를 기록합니다.

## 8. 서비스/예약 작업 비활성화

서비스와 예약 작업은 성능 및 개인정보 측면에서 조정할 수 있지만, 과도한 비활성화는 운영 장애로 이어질 수 있습니다.

운영 기준:

- `--lite` 모드로 시작합니다.
- 업무 앱과 보안 에이전트 설치 후 재검증합니다.
- Windows Update, Defender, 인증서, 시간 동기화 관련 항목은 무리하게 끄지 않습니다.
- 비활성화 후보는 `configs/services-disable-list.txt`, `configs/tasks-disable-list.txt`에서 관리합니다.

## 9. 삭제/비활성화 시 감사 대응 포인트

변경 이력에 다음을 기록합니다.

- 적용 모드
- 실행 스크립트 버전 또는 커밋
- 적용 일시
- 실행자
- 삭제한 Appx 목록
- 비활성화한 서비스/예약 작업 목록
- 적용한 정책 키
- 업무 앱 검증 결과
- Sysprep 결과
- 압축 및 복원 검증 결과

## 10. 권장 승인 흐름

1. 템플릿 작업자가 복제 VM에서 `--lite` 모드 실행
2. 운영 담당자가 앱/정책 영향도 검토
3. 보안 담당자가 개인정보/클라우드 기능 제한 항목 검토
4. `--standard` 또는 `--advanced` 적용 여부 결정
5. Sysprep 및 후처리 수행
6. 빌드 후 체크리스트 기록
7. 변경 이력 문서화
